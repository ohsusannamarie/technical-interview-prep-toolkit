# Technical Interview Prep by Role

Different technical interviews measure different things. Prep should match the role, the level, and the actual interview format.

Use this guide to focus your preparation instead of trying to study everything at once.

## How to use this guide

1. Find the role closest to the interview you are preparing for.
2. Review the focus areas.
3. Pick 1-2 projects or examples that prove those skills.
4. Practice explaining your decisions, tradeoffs, and impact out loud.

## Software Engineer

Focus areas:

- Data structures and algorithms
- Problem-solving communication
- Code readability
- Debugging
- Big O notation
- Testing and edge cases
- Past project impact

Prepare to discuss:

- How you break down ambiguous problems
- How you decide between approaches
- How you write maintainable code
- How you collaborate with product, design, QA, or other engineers

Good prep:

- Practice common coding patterns
- Review one or two projects deeply
- Practice explaining tradeoffs out loud

Strong signal:

- You can solve problems clearly, explain your assumptions, and write code that another human can understand.

## Frontend Engineer

Focus areas:

- JavaScript or TypeScript fundamentals
- UI architecture
- State management
- Accessibility
- Browser behavior
- Performance
- Testing
- Design collaboration

Prepare to discuss:

- Component design
- Data flow
- User experience tradeoffs
- Accessibility decisions
- Performance bottlenecks
- Cross-browser or responsive design issues

Good prep:

- Review recent frontend projects
- Be ready to explain how you structured components
- Practice discussing tradeoffs between usability, speed, and maintainability

Strong signal:

- You can connect technical implementation to user experience.

## Backend Engineer

Focus areas:

- APIs
- Databases
- Data modeling
- Distributed systems basics
- Caching
- Queues and messaging
- Reliability
- Security basics
- Observability

Prepare to discuss:

- API design choices
- Database tradeoffs
- How you handle failures
- How you scale services
- How you debug production issues

Good prep:

- Review system design fundamentals
- Prepare a project where you owned backend architecture or service design
- Practice explaining tradeoffs clearly

Strong signal:

- You can reason through scale, failure, data flow, and maintainability without pretending every system is magic.

## Full Stack Engineer

Focus areas:

- Frontend fundamentals
- Backend fundamentals
- APIs
- Data flow
- Product thinking
- Debugging across the stack
- Tradeoff communication

Prepare to discuss:

- How frontend and backend systems interact
- How you prioritize user experience and technical constraints
- How you debug issues across layers
- How you make implementation decisions with incomplete information

Good prep:

- Prepare one project that shows both product impact and technical depth
- Practice explaining the full flow from user action to data persistence

Strong signal:

- You can move between layers without losing sight of the user or the system.

## Data Engineer

Focus areas:

- SQL
- Python or Scala
- Data modeling
- ETL / ELT pipelines
- Warehousing
- Batch and streaming concepts
- Data quality
- Orchestration
- Performance optimization

Prepare to discuss:

- Pipeline design
- Data reliability
- Schema changes
- Monitoring and alerting
- Stakeholder requirements
- Tradeoffs between speed, cost, and accuracy

Good prep:

- Practice SQL problems
- Prepare examples of data pipelines you built or improved
- Be ready to explain how you handled messy data

Strong signal:

- You can build data systems people can trust.

## ML / AI Engineer

Focus areas:

- Python
- ML fundamentals
- Model evaluation
- Data preprocessing
- Feature engineering
- Experimentation
- Deployment basics
- Monitoring and drift
- Product impact

Prepare to discuss:

- How you chose a model or approach
- How you evaluated performance
- How you handled data quality
- How you moved from experiment to production
- How you communicated model limitations

Good prep:

- Prepare one project with clear problem, data, approach, tradeoffs, and results
- Review common ML metrics and when to use them
- Practice explaining technical work to non-technical stakeholders

Strong signal:

- You can explain not only what you built, but why it was the right approach for the problem.

## DevOps / SRE

Focus areas:

- Linux basics
- Networking fundamentals
- CI/CD
- Cloud infrastructure
- Containers
- Kubernetes basics
- Monitoring and alerting
- Incident response
- Reliability
- Automation

Prepare to discuss:

- How you prevent outages
- How you respond during incidents
- How you define and measure reliability
- How you automate repetitive work
- How you balance speed and stability

Good prep:

- Prepare incident response examples
- Review infrastructure decisions you have made
- Practice explaining root cause, mitigation, and prevention

Strong signal:

- You can keep systems healthy, explain failure clearly, and improve reliability without creating process theater.

## Hardware / Semiconductor Engineer

Focus areas will vary by specialty, but may include:

- Digital design
- Analog / mixed signal fundamentals
- Verification
- Physical design
- Timing closure
- Signal integrity
- Power, performance, and area tradeoffs
- Debugging methodology
- Cross-functional collaboration

Prepare to discuss:

- Technical fundamentals relevant to the role
- Specific tools, flows, or methodologies you have used
- A difficult debugging problem
- Tradeoffs you considered
- How you worked with adjacent teams
- How your work impacted schedule, quality, or performance

Good prep:

- Review the job description for domain-specific signals
- Prepare project examples with technical depth
- Be ready to explain your work clearly to both specialists and cross-functional partners

Strong signal:

- You can connect deep technical work to product, performance, quality, or schedule impact.

## Engineering Manager

Focus areas:

- People leadership
- Technical judgment
- Execution
- Prioritization
- Hiring and team building
- Stakeholder management
- Conflict resolution
- Strategy and delivery

Prepare to discuss:

- How you lead teams through ambiguity
- How you manage performance
- How you make technical tradeoffs
- How you partner with product or business leaders
- How you hire and develop talent
- How you handle conflict

Good prep:

- Prepare leadership stories with clear stakes and outcomes
- Be ready to explain your management philosophy
- Show how you balance people, product, and technical priorities

Strong signal:

- You can create clarity, make good tradeoffs, and help teams deliver without burning people into tiny professional croutons.

## Seniority lens

### Early career

Interviewers often look for:

- Fundamentals
- Learning agility
- Communication
- Coachability
- Problem-solving approach
- Curiosity

Best examples:

- School projects
- Internships
- Bootcamp projects
- Personal projects
- Volunteer or open-source work
- Times you learned something quickly

### Mid-level

Interviewers often look for:

- Independence
- Solid technical execution
- Collaboration
- Ownership of features or services
- Debugging ability
- Clear communication

Best examples:

- Features you owned
- Bugs you solved
- Projects you shipped
- Cross-functional work
- Process improvements

### Senior

Interviewers often look for:

- Technical judgment
- Ambiguity handling
- Architecture thinking
- Mentorship
- Cross-functional influence
- Tradeoff communication
- Ownership beyond assigned tasks

Best examples:

- Architecture decisions
- Mentorship moments
- Technical tradeoffs
- Leading through ambiguity
- Improving team or system outcomes

### Staff+ / Principal

Interviewers often look for:

- Systems thinking
- Strategic technical direction
- Influence without authority
- Organization-level impact
- Deep expertise
- Pattern recognition
- Ability to simplify complexity

Best examples:

- Multi-team impact
- Long-term technical strategy
- Platform or architecture work
- Technical alignment across groups
- Preventing or resolving systemic problems

## Universal prep questions

No matter the role, prepare answers to these:

- What is a project you are proud of?
- What was technically difficult about it?
- What tradeoffs did you make?
- What would you do differently now?
- How did you collaborate with others?
- How did you handle ambiguity?
- How did you measure success?

## The universal rule

Match your prep to the interview.

A backend system design interview, a frontend coding screen, a semiconductor debug conversation, and an engineering manager interview are not the same sport.

Prepare for the game you are actually playing.
