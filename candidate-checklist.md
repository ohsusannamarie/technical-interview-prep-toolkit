# Candidate Interview Prep Checklist

Use this as a practical pre-interview checklist. You do not need to do everything perfectly. The goal is to reduce ambiguity and walk in with a plan.

## 1. Confirm the basics

Before you start preparing, make sure you know:

- [ ] Interview date and time
- [ ] Time zone
- [ ] Interview length
- [ ] Interview format
- [ ] Interview platform or meeting link
- [ ] Names or roles of interviewers, if available
- [ ] Who to contact if the link breaks or something goes wrong

## 2. Understand the format

Ask or confirm whether the interview includes:

- [ ] Live coding
- [ ] Take-home assignment
- [ ] Pair programming
- [ ] Code review
- [ ] System design
- [ ] Architecture discussion
- [ ] Past project deep dive
- [ ] Behavioral interview
- [ ] Hiring manager conversation
- [ ] Recruiter screen

## 3. Clarify expectations

Try to understand:

- [ ] Which skills are being assessed
- [ ] Whether a specific programming language is required
- [ ] Whether you can use documentation
- [ ] Whether you need to share your screen
- [ ] Whether the interviewer cares more about speed, accuracy, communication, or tradeoffs
- [ ] Whether the role expects depth in a specific domain

## 4. Coding prep

Focus on patterns before random volume.

- [ ] Arrays and strings
- [ ] Hash maps and sets
- [ ] Two pointers
- [ ] Sliding window
- [ ] Stacks and queues
- [ ] Trees and graphs
- [ ] Recursion
- [ ] Sorting and searching
- [ ] Big O notation
- [ ] Edge cases

Practice saying your thought process out loud. Silent genius is hard to evaluate over Zoom.

## 5. System design prep

Be ready to talk through:

- [ ] Requirements
- [ ] Constraints
- [ ] Users and scale
- [ ] APIs
- [ ] Data model
- [ ] Storage choices
- [ ] Caching
- [ ] Queues or messaging
- [ ] Reliability
- [ ] Observability
- [ ] Tradeoffs

## 6. Behavioral prep

Prepare 3-5 stories you can adapt to different questions.

Strong stories usually include:

- [ ] Context
- [ ] Your role
- [ ] The problem
- [ ] What you did
- [ ] Why it mattered
- [ ] Result or lesson learned

Useful story themes:

- [ ] A hard technical problem
- [ ] A project you are proud of
- [ ] A mistake or failure
- [ ] A disagreement or conflict
- [ ] A time you influenced without authority
- [ ] A time you learned something quickly
- [ ] A time you improved a process, system, or team outcome

## 7. Project deep dive prep

Pick 1-2 projects you can discuss in detail.

For each project, know:

- [ ] What problem it solved
- [ ] Who used it
- [ ] Your specific contribution
- [ ] Key technical decisions
- [ ] Tradeoffs
- [ ] Challenges
- [ ] Metrics, results, or impact
- [ ] What you would improve now

## 8. Company and role prep

Review:

- [ ] Company website
- [ ] Product or platform
- [ ] Engineering blog, if available
- [ ] Recent news or announcements
- [ ] Job description
- [ ] Team or domain context
- [ ] Technologies mentioned in the role

## 9. Questions to ask

Prepare questions that help you evaluate fit.

Examples:

- What would success look like in the first 6 months?
- What are the biggest technical challenges facing the team right now?
- How does the team make architecture decisions?
- How do engineering, product, and design work together?
- What is the code review culture like?
- How is technical debt handled?
- What kind of person tends to thrive on this team?

## 10. Day-of checklist

Before the interview:

- [ ] Test camera and microphone
- [ ] Check Wi-Fi or hotspot backup
- [ ] Close distracting tabs
- [ ] Open your resume and notes
- [ ] Have water nearby
- [ ] Silence notifications
- [ ] Join 2-3 minutes early
- [ ] Take one actual breath before speaking

## Final reminder

You do not need to sound like a textbook. You need to be clear, collaborative, and honest about how you think.

A good interview is not a performance of perfection. It is a working session where people get to see your judgment.
