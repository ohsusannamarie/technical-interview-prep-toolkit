# Contributing

Thanks for helping improve this toolkit.

This repo is meant to stay practical, candidate-friendly, and easy for recruiters to share. It is not trying to become the largest technical interview prep list on GitHub.

## Good contributions

Helpful contributions include:

- Candidate-friendly resources
- Clear coding practice links
- System design resources
- Behavioral interview prep resources
- Role-specific prep guidance
- Recruiter enablement language
- Interview checklists
- Accessibility improvements
- Broken link fixes

## Resource criteria

Before suggesting a resource, ask:

- Is it useful for real candidates?
- Is it clear what skill it helps build?
- Is it free or reasonably accessible?
- Is it specific enough to be helpful?
- Would a recruiter feel comfortable sending it to a candidate?
- Does it reduce ambiguity instead of adding noise?

## What to avoid

Please avoid:

- Spammy links
- Affiliate-heavy resources
- Giant link dumps with no context
- Resources that require expensive subscriptions without noting that clearly
- Content that is outdated, misleading, or overly generic
- Advice that encourages candidates to memorize answers instead of understanding concepts

## Suggested format for new resources

When adding a resource, use this format:

```md
### Resource Name

https://example.com

Best for:

- Skill or use case
- Skill or use case
- Skill or use case
```

## Tone

Keep the tone:

- Clear
- Practical
- Human
- Encouraging
- Not robotic
- Not fear-based

Candidates are already carrying enough interview stress. The repo should help them feel more oriented, not more behind.

## Pull requests

For pull requests:

1. Keep changes focused
2. Explain what you changed and why
3. Include context for new resources
4. Check links before submitting
5. Avoid adding resources just to make the list longer

Quality beats quantity here.
